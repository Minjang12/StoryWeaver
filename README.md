# StoryWeaver

* 1-0. 개요

"작가의 사고 흐름을 끊지 않는다."라는 신조로 만든 스토리 창작 보조 프로그램입니다.
스토리의 노드화를 통해, 간단하게 텍스트 스토리를 창작하고 연결하는 데 특화되어 있습니다.

* 2-1. 프로젝트 생성
   
  **새 프로젝트 생성**: 프로젝트 관리 창에서 새 프로젝트 이름을 입력한 후 우측의 파란색 '+' 버튼을 클릭합니다.

텍스트 파일로 프로젝트 생성: 기존에 작성된 텍스트 파일(.txt)을 <스토리 위버> 포맷으로 변환하여 불러올 수 있습니다. 단, 현재 버전의 변환 시스템은 단순 텍스트 컨버팅 위주로 동작하므로 복잡한 문맥 서식이 완벽하게 반영되지 않을 수 있습니다.

  **파일에서 불러오기**: 기존에 <스토리 위버>를 통해 작업하고 저장했던 외부 프로젝트 파일을 직접 불러옵니다.

* 2-2. 캔버스
프로젝트 생성 시 화면 우측에 기본적으로 표시되는 작업 영역입니다. 초기 상태에서는 '인트로'라는 이름의 시퀀스 내에 'START'와 '두번째 씬' 노드가 생성됩니다.

  **화면 이동**: 마우스 휠 버튼을 클릭한 상태로 드래그하면 캔버스 영역을 자유롭게 탐색할 수 있습니다.

* 2-2-1. 씬 (Scene)
캔버스 위에서 개별 장면을 나타내는 사각형 노드입니다.

  **선택**: 씬을 클릭하면 파란색 테두리가 활성화되며 선택 상태가 됩니다.

  **시작 씬**: 'START' 문구가 표시된 씬은 프로젝트의 진입점이 되는 특별한 노드입니다. 변경이 필요한 경우, 원하는 다른 씬을 마우스 우클릭한 뒤 '시작 씬'으로 지정할 수 있습니다.

생성: 캔버스 빈 공간을 마우스 우클릭하고 '여기에 새 씬 추가'를 선택하거나, 화면 우하단의 파란색 '+' 버튼을 클릭합니다.

* 2-2-2. 시퀀스 (Sequence)
여러 개의 씬을 그룹화하여 관리할 수 있는 점선 형태의 상위 영역입니다.

  **설정 및 삭제**: 시퀀스 영역을 마우스 우클릭하여 배경 색상을 변경하거나 시퀀스를 삭제할 수 있습니다.

  **생성**: 빈 캔버스를 마우스 우클릭한 후 '여기에 새 시퀀스'를 선택합니다.

  **씬 편입**: 씬 노드를 마우스 우클릭한 후 '시퀀스에 추가' 메뉴에서 원하는 시퀀스를 선택합니다. 하나의 씬은 동시에 여러 시퀀스에 포함될 수 없으므로, 이미 시퀀스에 소속된 씬은 먼저 기존 시퀀스에서 제외해야 합니다.

* 2-3. 씬 편집기 (사이드바)
캔버스 좌측의 사이드바를 통해 선택한 씬의 상세 내용을 편집할 수 있습니다.

  **꾸미기 탭**: 씬의 색상과 아이콘을 변경하여 시각적으로 구분할 수 있도록 지원합니다.

  **기타 항목**: 각 탭 내의 '?' 아이콘에 마우스 커서를 올리면 툴팁을 통해 상세 사양을 확인할 수 있습니다. 최초 사용 시에는 등장인물 추가 및 대사 편집 기능부터 파악하는 것을 권장합니다.

* 2-3-1. 자동 연결 (다른 씬으로 연결)
분기 선택지 없이 스토리가 자연스럽게 다음 장면으로 이어지도록 설정하는 기능입니다.

  **사용 방법**: 씬 제목 옆에 위치한 사슬 모양 버튼을 클릭하면 노드 연결 모드가 활성화됩니다. 이 상태에서 이어 붙일 대상 씬을 클릭하면 점선 화살표로 두 노드가 자동 연결됩니다.

* 3-1. 타임라인 뷰
상단 헤더에 있는 '캔버스' 버튼을 클릭하면 '타임라인 뷰' 모드로 전환할 수 있습니다. 캔버스 뷰 형태에서 파악하기 어려운 사건의 시간 순서와 흐름을 시각적으로 일목요연하게 파악할 수 있습니다.

  **정합성 검사기**
캔버스 우상단의 정합성 검사 버튼을 실행하면 프로젝트 내부의 다양한 논리적 오류를 진단합니다. 보고서 우상단의 깔때기 모양 필터 버튼으로 원하는 항목만 검사할 수도 있습니다.

  **주요 검사 항목**:

    * 스토리 흐름상 절대 도달할 수 없는 변수 조건 검출
    * 상호 모순되는 변수 논리 조건 오류
    * 사망 또는 퇴장 처리가 되었음에도 다음 장면에 등장하여 대사를 출력하는 오류
    * 캐릭터에게 부여된 상태가 완결 시점까지 해결되지 않는 문제
    * 어떤 선택지를 고르든 결과 변수에 변화가 없고 동일한 씬으로 귀결되는 의미 없는 선택지 배치 오류

  **주의사항**: 현재 버전의 정합성 검사는 타임라인 뷰 모드 내에서만 실행을 지원합니다.

* 3-2. 상태 관리자
집필 과정에서 캐릭터에게 특정 상태(예: 사망, 행동 불능, 해결 추적 등)를 실시간으로 부여하여 추적하는 기능입니다. 대사 편집창이나 선택지의 '액션 추가' 메뉴를 통해 상태를 설정할 수 있습니다.

  **동작 원리**: 예를 들어 특정 캐릭터에게 '행동 불능' 상태를 적용해 두면, 이후 시점에서 해당 캐릭터가 다시 동작할 때 정합성 검사기가 이를 모순으로 판단하여 사용자에게 경고를 전달합니다. 스토리가 끝날 때까지 해결되지 않은 상태가 남아있는 경우에도 추적 오류를 보고합니다.

3-3. 플롯 스레드
씬 노드에 부여할 수 있는 일종의 태그 시스템입니다. 플롯 스레드 관리 모달이나 사이드바를 통해 태그를 자유롭게 생성하고 부착할 수 있습니다.

  **시각 효과**: 태그를 지정하면 캔버스 내 해당 씬 노드에 고유 색상이 표시되며, 노드 툴팁을 통해 설정된 태그 정보를 한눈에 확인할 수 있습니다. 시점이 교차하는 서사나 특정 캐릭터가 활약하는 플롯을 직관적으로 분류할 때 유용합니다.

* 3-4. 스토리 라인
다중 분기로 구성된 스토리라인 중 중심이 되는 메인 루트나 특정 조건의 루트를 명확히 구조화하는 기능입니다.

  **사용 방법**: 캔버스 우상단의 '스토리라인 관리' 모달을 통해 생성할 수 있으며, 포함될 씬들을 연결하면 순서대로 번호가 매겨집니다.

  **대본 보기**: 구축된 스토리라인은 '대본 보기' 기능을 통해 대본 형태로 열람할 수 있으며, 텍스트 복사 및 별도의 텍스트 파일 저장 기능을 지원합니다.

* 3-5. 캔버스 변수 정보 표시 및 숨기기
캔버스 우하단의 막대그래프 아이콘을 활성화하면 '캔버스 변수 정보 보기' 모드로 전환됩니다.

  **기능**: 현재까지 프로젝트 내에 생성, 지정, 연산된 변수들이 각 씬 노드 하단에 일괄 표시됩니다. 해당 모드 내에 자체 내장된 정합성 검사기가 변수의 계산 오류나 누락을 감지할 경우 붉은색 텍스트로 오류 내용을 상세히 안내합니다.

* 3-6. 통합 위키
작품 내 방대한 세계관, 고유 용어, 인물 설정 등을 체계적으로 기록하고 관리하는 데이터베이스입니다.

  **툴팁 연동**: 대사 편집 시 위키에 등록된 특정 단어가 입력되면 시스템이 이를 자동으로 인식합니다. 집필 중 해당 단어에 마우스 커서를 올리면 위키에 작성된 설정 내용이 툴팁으로 자동 출력되어 설정 오류를 방지할 수 있습니다.

* 3-6-1. 캐릭터 관계도
통합 위키 내부의 캐릭터 시트에 인물 간의 관계성을 기록해 두면, 시스템이 이를 분석하여 시각화된 '등장인물 관계도'를 자동으로 구성합니다.

  **주의사항**: 현재 버전에서는 캐릭터 프로필 이미지의 해상도 크기 및 정렬 위치가 일부 어긋나는 UI 현상이 존재합니다.

* 3-7. 스토리 분석
집필된 텍스트 데이터를 통계적 지표로 환산하여 보여주는 대시보드 기능입니다.

  **제공 데이터**: 지문(나레이션)과 대사의 구성 비율, 캐릭터별 대사 점유율 및 비중 추이, 인물 간 상호작용 및 대화 빈도, 스토리 진행에 따른 캐릭터 능력치 변동 그래프 등을 분석하여 제공합니다. 시스템 내부에는 창작 동기를 부여하기 위한 소소한 달성 과제(업적) 기능도 포함되어 있습니다.

* 4-0. 문제 해결 및 디버깅
  * 프로젝트 실행 중 프로그램이 예기치 않게 멈추거나 치명적인 시스템 오류가 발생하는 경우 아래 절차에 따라 디버그 정보를 제보해 주시기 바랍니다.
  * 오류가 발생한 상태에서 프로그램을 종료하지 마십시오.
  * 키보드의 F12 버튼을 눌러 개발자 도구(디버그 창)를 실행합니다.
  * 상단의 Console (콘솔) 탭으로 이동합니다.
  *출력된 로그 중 빨간색으로 표시된 오류 코드를 찾아 복사한 뒤 이슈 제보 채널을 통해 전달해 주시면 오류 수정 및 기능 개선에 반영하도록 하겠습니다.
